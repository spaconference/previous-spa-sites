SPA Conference session: Writing embedded Domain-Specific Languages in Java

One-line description:A hands-on workshop where participants learn and try out techniques for building DSLs
 
Session format: Workshop (150 mins) [read about the different session types]
 
Abstract:The use of Domain Specific Languages has become increasingly popular in recent times, with an aim of bringing the language in which we develop software closer to the language that our customers use when they talk about the business domain, and making code easier to read and clearer in intention.

There has been a good deal of activity recently, particularly in the Ruby community, in using dynamic languages to produce “embedded” or “internal” DSLs. Producing a similar effect in languages such as Java can appear more difficult.

In this hands-on session, developers will write their own embedded DSLs in Java, using techniques that we have learned from the design and evolution of embedded DSLs in projects such as LiFT, Hamcrest and jMock. We will collectively review the DSLs and discuss the benefits and drawbacks of writing embedded DSLs in Java. These techniques should be applicable to other Java-like languages, such as C#.

 
Audience background:Intended audience and experience level: developers - apprentice, journeyman or master.

Prerequisites: Attendees should bring a laptop with their preferred Java development environment installed. We will be working in pairs, so if you don’t have a laptop you should be able to pair up with somebody who does.
 
Benefits of participating:Attendees will learn some of the benefits of using DSL techniques in developing software, and come away with knowledge and practical experience of techniques for creating such a DSL in their own projects.
 
Materials provided:We will give attendees some existing code and tests and then get them to refactor towards an embedded DSL. We’re aware that time might be an issue so we will try to structure the exercise and introduction to give attendees enough hints that they can make some progress.
 
Process:Initial presentation of ideas and techniques (20 mins)
Attendees will work in pairs on laptops to develop an embedded DSL (60 mins) - we aim to provide two scenarios so that there is some variety amongst the group. Skeleton code and tests will be provided to get people started.
Swap pairs around and get experience of using an embedded DSL that somebody else has developed (30 mins)
In original pairs, discuss/refactor language based on feedback (20 mins)
Retrospective (20 mins)
 
Detailed timetable:00:00 - 00:20 Initial presentation
00:20 - 00:80 pairing on laptops to develop an embedded DSL
01:20 - 01:50 Swap pairs
01:50 - 02:10 discuss/refactor language based on feedback
02:10 - 02:30 Retrospective
 
Outputs:We will aim to collect examples produced by the participants during the workshop, and any lessons learned, and post them on a webpage.
 
History:This is a new session, but Robert Chatley has previously presented work on developing the LiFT Java DSL at the Google Test Automation Conference, and at XPDay 2006.
