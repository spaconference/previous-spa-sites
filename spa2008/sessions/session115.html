SPA Conference session: Managing Open Source in commercial products

One-line description:How to manage, constrain and stablise Open Source APIs for commercial projects
 
Session format: Case study (75 mins) [read about the different session types]
 
Abstract:In this session I discuss what management practices are needed to constrain and stablise an Open Source API for use in a commercial product.

The session draws on a case study of a specific Open Source API (DirectFB, for graphics and display mixing), that is used in commercial TV and Set Top Box products.

Open Source software is attractive in commerical products because it typically:

- adopts innovative design and rich sets of features
- develops rapidly to track evolving user needs

But the same richness of features and rapid development conflict with commercial product needs to have clearly focussed goals, and to be stable.

Issues I will discuss include:

- Usability: How to assess an Open Source API to see how suitable it is for a particular domain of application.

- Pruning: What to prune from a wide ranging Open Source API to produce a subset of functions that are focussed on the domain of application, are consistent, and that avoid repetition and redundancy.

- Managing change: Containing the problem of constant, and sometimes uncontrolled, change and development.

- Style: The importance of programming style and usage guidelines to formalise and constrain writing with OPen Source APIs.
