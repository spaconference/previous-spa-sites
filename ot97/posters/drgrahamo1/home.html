What is thoroughness in testing?

Test Coverage works like this:

  • First, count up the "things".
  • Then run the tests you have devised so far.
  • See whether your tests have exercised all of the "things" you identified.
  • If not, devise more tests until you cover them all (or how many you want to cover).

In traditional development, there are commercial tools which can measure the coverage of a number of different structural elements. The most commonly used is branch coverage (e.g. True and False outcome from IF) of code.

Even without tools, coverage concepts are extremely useful at many different levels of testing. For example, have you tested every requirement? Has every error message been seen? Has every screen appeared in at least one test?
