Architecture: what software people do now engineering is no 

longer fashionable.





Also:



Architecture: that which a complex artefact posesses by virtue of the 

properties of its components and connections as constrained by dictates 

of purpose, aesthetics, and culture.



Note:



* The definition holds for any artefact



* There is a complexity threshold: a mug doesn't have architecture,

a house or computer system does.



* Architecture is a resultant. Ill-formed and chaotic architectures tell us

as much about the culture that produced them as elegant, carefully-planned

ones.

When I was involved in the architecture teams for two massive projects (both of them OO based), we focused on the deliverables below :



1) Logical architecture

|----------------------GUI (OLTP) architecture

|----------------------Batch architecture

|----------------------Common framework



2) Process Architecture

3) Physical architecture



Item 2 focuses on the application server processes, their config info, error recovery, control record info, load balancing, scheduling  and the like.



Item 3 focuses on the physical network, machine capability and system management functions.



Item 1) is the logical modelling part. It focuses on :

- processes, inputs, outputs (captured by use cases and scenarios)

- common layers in the object framework (data type/location abstraction, utilities, visual framework, non-visual framework, class clusters and dependencies)



- We found that the dependency management was the most tricky part. To prevent tight coupling between abstraction layers in the framework, we had to resort to methods prescribed by Robert Martin (comp.object) and Grady Booch (book:Object Solutions)



I feel the patterns prescribed by the Gang-of-four in their book addresses Items 1) and 2) quite well. 



In a multi-process, multi-threaded environment, patterns like FLYWEIGHT may need to be applied based on the situation. We also realised that a new class of objects like Thread, Mutex, ResourceManager and DynamicCache got introduced. 



If you use the SanFrancisco framework (courtesy:IBM), the distributed object layer can be modelled to fit in with the rest of the framework quite easily
