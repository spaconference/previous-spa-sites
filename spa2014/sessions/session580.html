SPA Conference session: Armageddon: How to Survive the Changes Coming to Web Platform Development

One-line description: One-line description: The web ecosystem is currently undergoing massive changes, this session will tell you what you need to survive and give you a hands on of the action.
 
Session format: Tutorial 150 minutes [read about the different session types]
 
Abstract:Look at the big changes coming (some are already here) to development for the web platform: Webcomponents, Ecmascript6, sourcemaps, package management, in-browser development, http2. We’ll talk about what you need to know to adapt your practice and you’ll get hands on experience of some of the key changes.
 
Audience background:Some JS & Web development. Node and NPM installed. Git installed. Focus is on client side, but some parts are applicable to serverside too.
 
Benefits of participating:A better understanding of how to develop web applications for the coming years and how to set up a future friendly development environment. Enough knowledge to help transfer to the new web development reality.
 
Materials provided:Example code and starting points for each of the exercises.
 
Process:Three presentations from the front with two longer exercises. For the exercises, attendees will be in pairs.
 
Detailed timetable:00:00 - 00:20 Overview. Package management & webcomp.
00:20 - 01:10 Workshop: hands on packmanagement + webcomp dev
01:10 - 01:20 break
01:20 - 01:35 es6, source map, in browser dev
01:35 - 02:20 Workshop: hands on es6, in browser dev
02:20 - 02:25 http2, round up
 
Outputs:A self-built simple webcomponent using ecmascript 6 code.
 
History:This session will be new for SPA2014
