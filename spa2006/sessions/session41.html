SPA Conference session: Inversion of Control Containers in Java

One-line description:How IOC changes software application development and how to use IOC based on containers available for Java.
 
 Session materials
 
Session format: Workshop [read about the different session types]
 
Abstract:Today, business components typically run inside some sort of container, which provides the components with a standard runtime environment and a set of generally useful services. Containers help to standardise the form of business components and reduce redundancy between them, so helping components to be integrated to form applications.

J2EE and .NET both provide sophisticated containers for their components, residing within application servers. However, the complexity and development difficulties that such containers bring with them have recently led to a number of so-called "lightweight" or "inversion of control" (IoC) containers being developed, which allow Java components to be hosted less intrustively and with less development and runtime overhead.

This session will introduce the idea of application component containers and will answer a number of questions about IoC containers including the services that various containers provide, the benefits of using them and the limitations that they have.

The content of the session will include:

- A short history of IOC containers
- An explanation of the Inversion of Control pattern that the containers implement
- A discussion of the different ways to implement IOC (Constructor, setter etc.)
- A discussion of IoC when compared to its main alternative, the Service Locator
- A comparison of lightweight versus heavy containers (such as J2EE)
- An explanation of how such containers make testing simpler, by facilitating the
use of Mock Objects
- A discussionn of the limits of IOC containers (there is no silver bullet!)
- An introduction to some of the current containers.

The presentation material will be illustratedby two extensive practical exercises that will allow participants to get to grips with an IoC container implementation.

 
Audience background:To benefit most from the session, participants should have developed Java applications using J2EE containers and/or the service locator pattern. Against this background, the exercises allow participants to understand how IOC containers change the design of an application. Technically, participants should be proficient in Java and should bring a laptop with them to the session, ideally with a recent JDK and IDE installed.
 
Benefits of participating:- Understand when to use a lightweight or a heavyweight container.
- Understand the different flavours of IOC and their advantages and disadvantages.
- Understand how IOC containers can simplify the design of applications.
- Understand how IOC containers affect unit testing.
 
Materials provided:- Slides
- Software, including a number of Java IOC containers
- Exercises with detailed instructions, skeleton files and solutions

In the first exercise, a sample application will be deployed in an IOC containers. Different groups can use different containers and compare their in the discussion. The second exercise consists of the migration of a sample application to IOC and to compare differences in the design of the application before and after the migration.
 
Process:Following a presentation, participants will work on two exercises. After the exercises participants share their experiences in a discussion.
