SPA Conference session: Understanding Security with Patterns

One-line description:become a more security aware developer via security patterns
 
 Session materials
 
Session format: Tutorial [read about the different session types]
 
Abstract:According to security guru Bruce Schneier, security is all about trade-offs you take with respect to your always limited resources. Today, security is often a problem either when designing a system or-in the worst case-as an after-thought. Only a few experts have knowledge to design good security and we can safely assume that there is no security expert in most development teams. Before Design Patterns OO-design was a similar black art only performed well by experts. We expect Security Patterns to provide such leverage for designing secure systems, because they make discussions easier with the common names for concepts, make security design decisions more conciously, and because they tell the truth about their trade-offs.

This tutorial introduces the current work on security patterns:
* Why Security Patterns?
* What is a Security Patterns?
* History of Security Patterns
* Selected Security Patterns examples
* Community process, forum, and how to get involved
* Status and outlook - where will Security Patterns go?
 
Audience background:Interest or experience in security in a software related environment

Some understanding what patterns are and can provide for design

software developers, system architects, managers

Presenters
1. Peter Sommerlad
HSR Rapperswil

Peter Sommerlad is a software engineer in heart and soul. Since fall 2004 he has taught as professor for informatics at HSR Hochschule für Technik, Rapperswil. He heads the Institute for Software (http://www.ifsoftware.ch), that tries to improve software practice in Switzerland by the motto '''better software, simply faster'''. In addition to teaching and programming Peter writes patterns and shepherds other pattern authors. He is co-author of the new book on "Security Patterns". As co-author of "Pattern-oriented Software Architecture - A System of Patterns", aka POSA volume 1, Peter won the Software Productivity Award 1996. He is member of Hillside Group, Hillside Europe, the Swiss Software Engineering Network SWEN, ACM and IEEE Computer Society.
