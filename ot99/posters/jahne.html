A backoffice trading system with Gemstone

Alexander Jähne

Union Bank of Switzerland

This session represents an architecture overview of Union Bank of Switzerland's global treasury backoffice system. The system has been developed in Smalltalk and uses GemStone as its object-oriented database system. The system receives deals from the frontoffice and carries out typical backoffice processing, such as booking, paying, and confirmation of deals, matching incoming confirmation, and enabling backoffice staff to validate and change deal contracts. The goal of the project is to replace a suite of backoffice systems with a customisable system that can be installed in many trading branches of the bank.

Key issues addressed in the project are increasing automation and reducing maintenance cost. The volume of transactions is expected to increase to 20,000 deals a day, each deal having a complex data structure of many objects and references to other bank data in existing relational databases.

Object-oriented development started in February 1995 with a first version of the application running in production since September 1995. This first version used ObjectStore as the object-oriented database system. Development work is distributed between teams in Zurich and London.

The case study discusses the real-life experiences of the project and the lessons learned in implementing a large, scaleable, workflow-based business application using object technology.

Key topics

Workflow processing

Subsystem architecture of independent collaborating processes

Inter-process communication

Experience of using the active object-oriented database system GemStone

Comparing GemStone with a relational database system and with ObjectStore

Error handling

Interfacing to the outside world

Designing for scalability

Alexander Jähne is a database architect at Union Bank of Switzerland in Zurich, and has been closely involved in the backoffice system development project discussed in this case study session since the project started early in 1995. He has contributed a section to the book Object-oriented databases in practice, edited by M Loomis and A Chaudhri.
