Using patterns in large distributed systems

David Crone

ECsoft

This session presents a case study involving a large-scale distributed system development by a team experienced in such developments and aware of many of the more widely-published design patterns.

The session maps the usage of design patterns from printed page to deployed system. It also examines the lessons that can be learnt from these experiences, and what actual benefits can be expected from using design patterns in such an environment.

Key topics

The large-scale distributed system environment and its architecture

Use of the Presentation-Abstraction-Control (PAC) pattern to de-couple business process logic from GUI components

Increasing the effectiveness of the de-coupling by modifying and extending the PAC pattern using other patterns

Effect of the pattern on the development environment

Effect of the development environment on the pattern

Reuse of components developed with the aid of design patterns

Observed usage pattern for the design patterns in the development

Comparison of purported and actual benefits of pattern-usage

Rules of engagement for using patterns for maximum benefit in distributed application development

David Crone has fifteen years' experience in IT development largely in the development of distributed systems, including the design and implementation of IBM's OS/2 LAN Server and CICS on AIX products. David's experience spans programming and design of distributed systems using tools such as Open Group DCE through to distributed object technology analysis, design, and implementation. Over the last two years, David has been working for the ECsoft Group as technical architect on a number of distributed object technology projects providing enterprise-class commercial systems for the financial services, retail, and leisure industries.
