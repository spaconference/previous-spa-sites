A paradigm shift in teaching object technology

Jutta Eckstein

Integral Development

Discussions at conferences (for example, the Educators' Symposium at OOPSLA) have shown that we still feel it is difficult to teach object technology. This is true wherever the education takes place: it is hard to teach object technology in industry and in academia.

It is commonly believed that teaching object technology is hard because the students have to make a paradigm shift. This is true, especially when the students have already been educated in structural thinking and programming. But it is only half the truth. If the people dealing with analysis, design, and programming have to make a paradigm shift to learn object technology, it should also be necessary for the educators to make the paradigm shift in teaching object technology. The way structural thinking has been taught does not work for teaching object technology. We need to change the way we teach.

Picking up object technology is difficult for students because the educators still have not made the paradigm shift.

Jutta Eckstein works for Integral Development in the areas of training and consulting. She has been developing object-oriented software since 1990 and worked for ParcPlace Systems (later ParcPlace-Digitalk) for many years as both a trainer and a consultant. Before becoming an engineer, Jutta completed a pedagogical study to become a teacher.
