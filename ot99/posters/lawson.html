Writing improved interfaces

Julian Lawson

Quidnunc

This session provides guidance and patterns relating to writing interfaces using IDL. The objectives of the session are to identify common problems encountered when writing IDL, to provide patterns to make IDL creation easier, and explain how to identify and assess the quality of components defined using IDL.

Key topics

IDL and its use

- Distributed object architectures

- Component software

- Communicating design

- Defining software contracts

Forces involved in interface definition

- Granularity

- Flexibility

- Extensibility

- Lifetime

- Dependencies

- Encapsulation

How interface design can affect these forces

How to make a compromise

A pattern catalogue: for example, role-based interfaces, private interfaces, pass by value, tokens, avoiding inheritance

Summary

Julian Lawson is the Practice Leader in Object Technology at Quidnunc. He has been working in the field of object technology for six years. At Quidnunc, he has been leading the move towards component architectures, designing object-oriented frameworks, and working on the Scenario Modelling methodology.
