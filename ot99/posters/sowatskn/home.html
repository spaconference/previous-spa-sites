Introduction

Java Beans is the object model for Java. It provides a standard API that can be used for defining reusable software components that can be manipulated visually or programmatically by other software, including builder tools and code generators.

All Java objects can be introspected by the Introspection API to determine their properties: attributes, produces events and consumed events. Attributes are determined by the presence of getX() and setX() methods, produced events by the presence of methods that register Listenersand consumed events by determing which Event Listener interfaces are implemented.

Any class can be a Bean, as are all of the classes in the JDK1.1, by following the standards defined for the APIs. In addition though, objects that have not been wriiten to these, standards can also be made into Beans by the addition of BeansInfo and, if required, specialised Property Editor classes.

Who Should Attend?

The workshop will be relevant to Java developers of any level of experience. Previous knowledge of Java Beans is not assumed. If you have experience of other component and object models (such as COM/ActiveX) you will find it useful to compare the features of Java Beans.

Workshop Structure

This workshop will cover:
  • Java Beans basics:
    • The Java Beans object model and the role of Introspection
    • Event handling
    • Bound Properties
    • Property Editors
    • JAR Files
     
  • Advanced Java Beans:
    • Activation Framework
    • InfoBus
    • Java Beans and CORBA
Presentations of key aspects of the API will describe Java Bean's features and explain the thinking behind the component object model. These will alternate with individual and group exercises where example beans will be created for use in various scenarios. 82
