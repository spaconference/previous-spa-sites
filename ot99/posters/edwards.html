Distribution in Smalltalk and Java systems

Steve Edwards, Matthew Bradley

The Object People

Most of the attention relating to distributed object systems these last few years has been focused on middleware adhering to the COBRA specification.

However, if a development requiring processing on multiple platforms can be implemented using only one language, alternative and essentially simpler approaches are available. This workshop provides an opportunity for hands-on experience of using IBM Distributed Smalltalk, Java RMI, and one of a number of other frameworks that are becoming available (for example, Caltech's Java-based InfoSpheres. A simple distributed object system will be implemented, with the intention of exploring basic techniques and implementation issues, comparing systems, and presenting the findings to participants.

Key topics

Working with and exploring the in-built distributed features of Smalltalk or Java

Implement a small model with simple views distributed to different platforms

Comparison of the capabilities of different systems and determination of their relevance in differing environments

Steve Edwards is Managing Director of The Object People. He has been directly involved in a variety of large corporate Smalltalk-based industrial projects since 1987. Has also trained programmers across the world in the arts of object-oriented development in a variety of Smalltalk and visual programming tools.

Matthew Bradley is a consultant and trainer with The Object People. He works primarily with the company's Java team, working in Europe and North America to help customers build Java solutions. He has worked on a variety of projects including one to enable family doctors to access hospital information using Java technology. Areas of expertise include visual programming with VisualAge for Java.
