Java Beans

Introduction

Java Beans is the component object model for Java. It provides developers with the means for creating reusable software components that can be manipulated visually or programmatically by other software, including builder tools and code generators.

Java objects can be introspected by the Introspection API to determine their properties, attributes, produced events and consumed events. Attributes are determined by the presence of getX() and setX() methods, produced events by the presence of methods that register Listeners and consumed events by determining which Event Listener interfaces are implemented.

Any class can be a Bean by implementing the APIs as described in the Java Beans specification. In addition, objects that have not been written to these standards can also be used as Beans if user defined BeansInfo classes are provided.

Who Should Attend?

The workshop will be relevant to Java developers of any level of experience. Previous knowledge of Java Beans is not assumed. If you have experience of other component and object models or (such as COM/ActiveX) you will find it useful to compare the features of Java Beans.

Workshop Structure

This workshop will cover:

  • What is a Java Bean?
  • Introspection
  • Properties
  • Event handling
  • Actions
  • Persistence
  • Customization
  • Design time/Run time
  • Security
  • Local/Distributed Beans
  • Concurrency

The workshop will combine the slides with exercises to develop a collection of Java Beans to illustrate key points as they are presented.
